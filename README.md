# Templatr Catalog

Community-contributed prompt templates for [Templatr](https://github.com/josiahH-cf/templatr).

## What is this?

The Templatr Catalog is a public, static JSON index of community prompt templates. Templatr users can browse, preview, and install templates from this catalog directly in the app via the `/browse` command — no API keys, no cloud account, no sign-up.

## Browsing the Catalog

In Templatr, type `/browse` in the chat bar to open the catalog browser. From there you can:

- Search templates by name, description, author, or tags
- Filter by category tag
- Preview template details before installing
- Install a template to your local library with one click

The app fetches the catalog from:

```
https://raw.githubusercontent.com/josiahH-cf/templatr-catalog/main/catalog.json
```

## Catalog Format

`catalog.json` is a flat JSON array. Each entry has these required fields:

| Field          | Type     | Description                                        |
|----------------|----------|----------------------------------------------------|
| `name`         | string   | Display name of the template                       |
| `description`  | string   | One-sentence description                           |
| `author`       | string   | GitHub username or display name of the contributor |
| `tags`         | string[] | Category tags (lowercase, hyphenated)              |
| `version`      | string   | Semantic version string (e.g. `"1.0.0"`)           |
| `download_url` | string   | Raw URL to the template `.json` file               |

Example entry:

```json
{
  "name": "Code Review Checklist",
  "description": "Systematic review prompt covering correctness, security, and tests.",
  "author": "your-github-handle",
  "tags": ["code", "review", "engineering"],
  "version": "1.0.0",
  "download_url": "https://raw.githubusercontent.com/josiahH-cf/templatr-catalog/main/templates/code_review_checklist.json"
}
```

## Template File Format

Each template file under `templates/` must be a valid Templatr template JSON:

```json
{
  "name": "Template Name",
  "description": "What this template does",
  "trigger": ":optional_trigger",
  "content": "Your prompt content with {{variable}} placeholders",
  "variables": [
    {"name": "variable", "label": "Label shown in UI", "multiline": false}
  ]
}
```

Required fields: `name`, `content`. All others are optional but recommended.

## How to Contribute a Template

1. **Fork** this repository.
2. Add your template file under `templates/` (filename: `your_template_name.json`, lowercase snake_case).
3. **Test your template locally** first: in Templatr, use `/import` to import your `.json` file and verify it works as expected.
4. Add a corresponding entry to `catalog.json` with the correct `download_url` pointing to your file on `main`.
5. Open a **pull request** with:
   - A brief description of what the template does and who it is for.
   - Confirmation that you tested it locally with Templatr.

`catalog.json` is also auto-regenerated by CI on every merge to `main`, so if your PR adds the template file but forgets to update `catalog.json`, the action will catch it.

### Validation checklist

Before submitting, make sure your template:

- [ ] Has a `name` and `content` field
- [ ] Uses `{{variable_name}}` syntax for dynamic input
- [ ] Has a clear, one-sentence `description`
- [ ] Includes at least one `tags` value
- [ ] Is tested with `/import` in Templatr and generates useful output

## Hosting a Private Catalog

You can point Templatr at any raw JSON endpoint that returns a valid catalog array. To use your own catalog:

1. Fork this repo (or create a new repo with the same structure).
2. In Templatr, go to **LLM Settings** and change the **Catalog URL** field to the raw URL of your `catalog.json` (e.g. `https://raw.githubusercontent.com/your-org/your-catalog/main/catalog.json`).
3. Use `/browse` as normal — Templatr will fetch from your URL instead.

This works for company-internal template libraries, team collections, or personal template sets.
